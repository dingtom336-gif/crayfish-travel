# 小龙虾旅行反向撮合产品需求文档（PRD）

## 文档信息
- 文档版本：V1.5（最终完整版）
- 适用场景：寒暑假旅行反向竞价、整单锁库存、双支付入口、全额无损退、实名合规收集
- 核心约束：中国境内航旅实名规则，必须先收身份信息再锁真实库存
- 适用渠道：PC网页、IM机器人、小龙虾技能端

## 一、产品定位

本技能是 AI 驱动的寒暑假专属旅行反向撮合服务。

用户提交固定预算与出行需求 -> 系统在暑期/寒假非高峰日期内，向 OTA 自营及入驻旅行社招标 -> 一次性锁定 5 套真实可履约全包套餐（机票+酒店+门票+100 元无损退保险）-> 每套展示供应商信息、15 分钟锁价、支付二维码+支付宝口令。

用户任选一套付款，未选自动释放；付款后订单可在淘宝/支付宝体系内管理，全程解决库存不同步、退改风险、匿名无法履约、信息收集敏感等问题。

## 二、用户核心需求与痛点

### 2.1 用户需求
- 仅限定暑期/寒假出行，接受非高峰日期，不追求高峰低价
- 固定总预算全包，含交通、酒店、必玩景点、保险
- 支付方式双兜底：二维码 + 支付宝口令，防止图片屏蔽
- 15 分钟锁价，从容对比 5 套方案
- 明确展示供应商主体（OTA 自营/旅行社），提升信任
- 付款后有可管理订单，可售后、可退款
- 身份信息收集方式温和、不尴尬、不泄露隐私

### 2.2 核心痛点
- 分次购买导致机票/酒店/门票库存不同步，下单易失败
- 只比价不展示主体，低价不信任，不敢下单
- 退改扣费严重，用户决策压力大
- IM 环境二维码易被屏蔽，无法支付
- 一上来索要身份证，用户敏感抵触
- 匿名付款后找不到订单、找不到商家
- 不实名无法锁定真实库存，占位无效

## 三、刚性规则（开发必须遵守）

### 3.1 时段规则
- 仅支持：暑期（7.1-9.30）/ 寒假（1.15-2.28）
- AI 自动剔除高峰，不提供以下日期方案：
  - 暑期：7.15-8.15
  - 寒假：腊月廿三 ~ 正月十五
- 仅在非高峰、尾单日期内匹配可行方案

### 3.2 预算与保险规则
- 总预算全包：往返交通 + 酒店 + 必玩门票 + 100 元全额无损退保险
- 保险权益：任意时间退改，全额退款无手续费
- 前端必须拆分显示：总价 = 基础费用 + 100 元保险

### 3.3 供应商规则
- 类型 A：OTA 平台自营（飞猪/携程/美团自营等）
- 类型 B：OTA 入驻旅行社（XX 国旅·飞猪店等）
- 必须支持：整单锁 15 分钟、实名出票、对接淘宝/支付宝订单、保险理赔

### 3.4 库存锁价规则
- 必须先收集身份信息，再锁库存，否则无法获取真实可出票仓位
- 每套方案一次性锁定：机票 + 酒店 + 门票 + 保险额度
- 生成二维码/口令 = 锁价成功，严格 15 分钟有效期
- 最终仅展示 5 套满足条件方案，不多不少

### 3.5 支付规则
- 每套方案：1 个总支付二维码 + 1 条支付宝付款口令
- 用户仅支付 1 套即可，其余 4 套超时自动取消释放
- 支付即完成订单建档，支持售后与退款

## 四、身份信息收集机制（前置、合规、不尴尬）

### 4.1 收集时机
技能启动第一步，AI 锁库存之前必须完成，否则无法向航司/酒店/景区发起真实锁单。

### 4.2 收集内容（仅必填最少项）
- 称呼（可选，先生/女士即可）
- 手机号（接收行程通知、酒店预留）
- 出行人姓名（实名）
- 证件号码（身份证/护照，用于机票/门票/保险实名）

### 4.3 前端引导文案
> 为了帮你锁定真实可出票的机票、酒店、门票，避免无效占位，需要填写以下信息。
> 所有内容仅用于本次锁单与履约，使用后自动删除，绝不外泄。

### 4.4 存储规则
- 信息加密存储
- 仅用于本次 5 套方案锁单与出票
- 订单支付完成/超时取消后自动清除

## 五、完整产品流程

1. 用户启动小龙虾旅行技能
2. 系统温和收集身份信息（姓名+手机+证件号）
3. 用户输入自然语言需求
4. AI 解析需求，剔除寒暑假高峰，判断可行性
5. 向 OTA 自营、旅行社商户发起统一招标
6. 系统批量锁定 5 套真实全包方案（交通+酒店+门票+保险）
7. 前端展示 5 套方案卡片，包含价格、供应商、倒计时、二维码、口令
8. 用户选择支付方式：
   - 扫码支付 -> 淘宝授权登录 -> 订单落入淘宝"我的订单"
   - 口令支付 -> 支付宝支付 -> 订单挂载供应商小程序
9. 支付成功 -> 供应商自动履约（出票、订房、预约、发送行程单）
10. 未支付方案 -> 15 分钟后自动释放库存，无任何费用
11. 退款：凭 100 元保险，随时全额无损退回


## 七、需要调用的 API 类型

1. NLP 需求解析 API
2. 寒暑假日期筛选 & 高峰剔除 API
3. 机票实时查询 + 锁舱 API
4. 酒店实时查询 + 锁房 API
5. 景区门票锁票 API
6. 无损退保险投保 & 额度锁定 API
7. OTA 自营供应商报价 API
8. 旅行社商户报价 & 资质 API
9. 支付宝二维码生成 API
10. 支付宝付款口令生成 API
11. 淘宝开放平台授权登录 API
12. 淘宝订单创建 & 回调 API
13. 支付宝小程序挂载 API
14. 15 分钟倒计时 & 自动释放 API
15. 用户信息加密 & 自动销毁 API

## 八、关键逻辑强调（产品底线）

- 在中国，不实名绝对锁不了真实机票/酒店/门票，信息必须前置收集
- 以"锁定真实库存"为理由收集信息，用户不尴尬、不抵触
- 二维码+口令双支付兜底，任何环境均可支付
- 供应商必须明示，提升信任，避免纯价格竞争尴尬
- 100 元保险降低决策压力，实现"随便买、随便退"
- 订单落入淘宝/支付宝，可管可查，不再失联
